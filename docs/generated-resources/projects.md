## 1. Add the Prisma model

Add the following to `app/schema.prisma` inside the `model User` block
(append to the existing relation fields):

```prisma
  projects Project[]
```

Then add the model definition anywhere in `app/schema.prisma`:

```prisma
model Project {
  id        String   @id @default(uuid())
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  user      User     @relation(fields: [userId], references: [id])
  userId    String
  name      String
  notes     String?

  @@index([userId])
}
```

## 3. Add a navigation item

In `app/src/client/components/NavBar/constants.ts`, add the new route to
`demoNavigationitems` (and/or `marketingNavigationItems` if public):

```ts
{ name: "Projects", to: routes.ProjectsRoute.to },
```

## 4. Run the database migration

```bash
cd app
wasp db migrate-dev --name add_projects
```

---

## 5. Start the dev server and verify

```bash
cd app
wasp start
```

Manual test checklist:

- [ ] Navigate to `/projects` while logged in — the page loads.
- [ ] The empty state is shown with a "New project" button.
- [ ] Create a new `project` — it appears in the list.
- [ ] Edit the `project` — changes persist.
- [ ] Delete the `project` — it is removed.
- [ ] Search filters the list correctly.
- [ ] Log in as a different user — the first user's `projects` are not visible.
- [ ] Direct API calls to update/delete another user's `project` return 404.
