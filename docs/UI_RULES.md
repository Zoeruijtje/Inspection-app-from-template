# UI Rules

## Style

Use the existing Open SaaS/shadcn/Tailwind style. Do not introduce a second UI system.

## Page patterns

Every resource should have standard pages:

- List page
- Create page
- Detail page
- Edit page
- Optional settings page

## Components to prefer

- Dashboard shell
- Data table
- Search/filter bar
- Form card
- Empty state
- Confirmation modal
- Toast notifications
- File dropzone
- Status badge

## UX rules

- Forms must show loading states.
- Forms must show validation errors.
- Destructive actions must ask for confirmation.
- Empty states must explain what to do next.
- Tables must support basic search/filter where useful.
- Do not create visually impressive but unusable screens.
